python文件内容操作

做题的时候碰到需要读取文件的操作了，于是准备学习一下相关的知识

基础知识

文件按照数据的组织形势分为两种：1）文本文件、2）二进制文件

文本文件

文本文件存储的是常规字符串，由若干文本行组成，通常每行以换行符’\n’结尾。常规字符串是在文本编辑器中可以正确显示、编辑，并且人类可以直接阅读理解的字符串。实际上在磁盘中，文本文件也是以二进制的形势存储的，只是在读取查看的时候使用了正确的编码方式进行了解码，将二进制信息还原为了字符串信息，所以可以直接阅读和理解。

二进制文件

二进制文件常见的有图形图像文件、音视频文件、可执行文件、资源文件、各类数据库文件 、各类Office文件等。这种文件不能被记事本或其他普通字处理软件编辑，需要有正确的软件进行解码或者反序列化才能正确的读取。

模式 说明
r 读模式（默认模式，可省略），如果文件不存在则抛出异常
w 写模式，如果文件已存在，先清空原有内容
x 写模式，创建新文件，如果文件已存在则抛出异常
a 追加模式，不覆盖文件原有内容
b 二进制模式（可与其它模式组合使用），使用二进制模式打开文件时不允许指定encoding参数
t 文本模式（默认模式，可省略）
+ 读写模式（可与其它模式组合）

“只读”和“只写”模式打开时文件指针的初始位置是文件头，“追加”模式打开时文件指针的初始位置为文件尾，“只读”模式打开的文件不能进行写操作，“只写”模式也不能进行读操作。

如果执行正常，open（）函数会返回一个可迭代的文件对象，通过该对象可以对文件进行读写操作，如果指定文件不存在、访问权限不够、磁盘空间不够或其它原因导致创建文件对象失败则抛出异常。当对文件内容操作完成后，一定要关闭文件对象（f1.close()），这样才能将修改保存。当然，即使写了关闭文件的代码，也不能保证文件就一定可以正常关闭（with关键字可以避免这个问题）

文件对象属性与常用方法

文件对象属性

属性 说明
buffer 返回当前文件的缓冲区对象
closed 判断文件是否关闭，若文件已关闭则返回Trued
fileno 文件号
mode 返回文件的打开方式
name 返回文件的名称

常用方法

方法 功能说明
close() 把缓冲区的内容写入文件，同时关闭文件，并释放文件对象
flush() 把缓冲区的内容写入文件，但不关闭文件
read([size]) 从文本文件中读取size的字节（Python2.x）或字符(Python3.x)的内容作为结果返回，或从二进制文件中读取指定数量的字节并返回，如果省略size则表示读取所有内容
readline() 从文本文件中读取一行内容作为结果返回
readlines() 把文本文件中的每行文本作为一个字符串存入列表中，返回该列表，对于大文件会占用较多内存，不建议使用
seek(offset[,whence]) 把文件指针移动到新的位置，offset表示相对于whence的位置。whence为0表示从文件头开始计算，whence为1表示从当前位置开始计算，whence为2表示从文件尾开始计算，默认为0
seekable() 测试当前文件是否支持随机访问，如果文件不支持随机访问，则调用方法seek()、tell()和truncate()时会抛出异常
tell() 返回文件指针的当前位置
write(s) 把字符串s的内容写入文件
writelines(s) 把字符串列表写入文本文件，不添加换行符

文件读写操作相关的函数都会自动改变文件呢指针的位置，例如以读模式打开文本文件，读取10个字符，会自动把文件指针移动到第11个字符的位置，再次读取字符的时候总是从文件指针的当前位置开始，写入文件的操作函数也具有同样的特点

问题总结

报错“embedded null character”

这个通常是因为路径里面的“\”被当做转移字符了，只要将路径字符串前面加上r就可以，或者把“\”换成“\”

报错“unsupported operand type(s) for pow(): ‘bytes’, ‘mpz’, ‘int’”

这个问题来自于pow函数支持的数据类型，因为在计算逆元的时候返回的结果是mpz，这个可以用int()来强制转换。

报错“’gbk’ codec can’t decode byte 0xd7 in position 13: illegal multibyte sequence”

这个问题应该是说编码的格式不对，在指定读取模式的后面加上encoding=”utf-8”

报错“an integer is required (got type str)”

这个问题是在指定编码方式时没有加上encoding=。如果不加上，那么默认的第三参数是buffering，该参数本身需要传入一个整型变量才行。

格式为def open(file, mode=’r’, buffering=None, encoding=None, errors=None, newline=None, closefd=True):

报错“SyntaxError: invalid character in identifier”

意思是标识符中存在无效字，很有可能是因为复制代码时出现了中文字符导致了无法识别或者多余的空格出现在行首或者行尾。

报错“ValueError: invalid literal for int() with base 10”

意思是以十为基数的int文本无效

报错“TypeError: ‘builtin_function_or_method’ object is not subscriptable”

问题可能出现在函数括号用错了，比如本来是应该用( )，结果用了[ ]
