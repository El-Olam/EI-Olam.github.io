python rsa模块的使用

生成密钥

函数rsa.newkeys(密钥大小)可以用来创建密钥对（生成公钥和私钥），并返回一个元组 ( rsa.PublicKey, rsa.PrivateKey)

类 rsa.PublicKey( n: int , e: int )

表示公共 RSA 密钥。

此密钥也称为“加密密钥”。它包含“n”和“e”值。

支持属性以及类似字典的访问。不过，属性访问速度更快。

类 rsa.PrivateKey( n: int , e: int , d: int , p: int , q: int )

表示私有 RSA 密钥。

此密钥也称为“解密密钥”。它包含“n”、“e”、“d”、“p”、“q”等值。

支持属性以及类似字典的访问。不过，属性访问速度更快。

生成密钥对可能需要很长时间，具体取决于所需的位数。位数决定了密钥的加密强度，以及您可以加密的消息的大小。如果您不介意使用比您要求的稍小的密钥，您可以通过accurate=False以加快密钥生成过程。

另一种加速密钥生成过程的方法是并行使用多个进程来加速密钥生成。使用的进程数不要超过您的机器可以并行运行的进程数；双核机器应该使用poolsize=2；四核超线程机器可以在每个核上运行两个线程，因此可以使用poolsize=8.

如果密钥生成太慢，还可以使用 OpenSSL 生成，然后将它们加载到Python 代码中。OpenSSL 在 3.5 秒内在上面使用的同一台机器上生成一个 4096 位的密钥。有关更多信息，请参阅与 OpenSSL 的互操作性。

加密与解密

加密消息：rsa.encrypt()

解密消息：rsa.decrypt()

假设 Alice 想要发送一条只有 Bob 可以读取的消息。

  1. Bob 生成一个密钥对，并将公钥提供给 Alice。这样做是为了让 Alice 确定密钥确实是 Bob 的（例如通过交出包含密钥的 U 盘）。

  2. Alice 编写了一条消息，并将其编码为 UTF-8。RSA 模块只对字节进行操作，而不对字符串进行操作，因此这一步是必须的。

  3. Alice 使用 Bob 的公钥加密消息，然后发送加密的消息。

  4. Bob 收到消息，并用他的私钥解密。

由于 Bob 将他的私钥保密，Alice 可以确定他是唯一可以阅读消息的人。但鲍勃不能确切知道它是爱丽丝发送消息，因为她没有签字。

RSA 只能加密小于密钥的消息。随机填充会丢失几个字节，其余字节可用于消息本身。例如，一个 512 位的密钥可以编码一个 53 字节的消息（512 位 = 64 字节，11 字节用于随机填充和其他内容）。有关如何处理较大文件的信息，请参阅处理大文件。

签名和验证

签名函数

函数rsa.sign(message: bytes ,priv_key: rsa.PrivateKey ,hash_method: str )可以为消息创建hash签名

这个函数利用私钥对消息进行签名

参数：

  • message – 要签名的消息。可以是 8 位字符串或类似文件的对象。如果message有一个read()方法，则假定它是一个类文件对象。
  • priv_key –rsa.PrivateKey要签名的
  • hash_method – 消息上使用的哈希方法。使用“MD5”、“SHA-1”、“SHA-224”、“SHA-256”、“SHA-384”或“SHA-512”。

返回：消息签名块（bytes）

溢出错误- 如果私钥太小而无法包含请求的散列。

函数rsa.verify(message: bytes , signature: bytes , pub_key: rsa.PublicKey )验证签名是否与消息匹配，该方法可以从签名中自动检测散列方法。如果验证成功，此函数返回 True：

处理大文件

RSA 只能加密小于密钥的消息。随机填充会丢失几个字节，其余字节可用于消息本身。例如，一个 512 位的密钥可以编码一个 53 字节的消息（512 位 = 64 字节，11 字节用于随机填充和其他内容）。

对较大文件使用 RSA 的最常见方法是使用 AES 或 DES3 等块密码用随机密钥加密文件，然后用 RSA 加密随机密钥。您可以将加密文件和加密密钥一起发送给收件人。完整的流程是：

  1. 生成随机密钥

  2. 使用该密钥通过 AES 加密文件。

  3. Encrypt 带有 RSA 的 AES 密钥

  4. 将加密文件与 encrypted_aes_key

  5. 收件人现在反转此过程以获取加密文件。

注：Python-RSA 模块不包含为您执行 AES 加密的功能。
